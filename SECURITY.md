# Security Policy

## Supported Scope

Security issues are most relevant for the currently active code on the main branch.

This includes, in particular:

- authentication-related code
- runtime and deployment configuration
- database access patterns
- scripts that handle league, roster, transaction, or reconciliation data

## Reporting a Vulnerability

Please do **not** open a public issue for suspected security vulnerabilities.

Instead:

1. Use GitHub private vulnerability reporting for this repository, if enabled.
2. If private reporting is not enabled, contact the repository owner privately through GitHub.

## What to Include

Please include as much of the following as possible:

- affected file(s) or script(s)
- clear description of the issue
- reproduction steps
- impact assessment
- any proposed mitigation or fix

## Response Expectations

I will review legitimate security reports as time allows and prioritize issues based on severity, exploitability, and impact.

## Out of Scope

The following are generally out of scope unless a clear security impact is demonstrated:

- general setup questions
- feature requests
- non-security code quality issues
- issues caused by local misconfiguration outside the repository
